# Repo Instructions

- When a task changes user-visible behavior, guide content, checker coverage, UI flow, or evaluation logic, update `CHANGELOG.md` before finishing.
- Do not add changelog entries for formatting-only cleanup, dependency-free internal refactors, or no-op maintenance that does not change behavior.
- Keep changelog entries concise, gameplay-relevant, and focused on what changed, why it mattered, and how it affects farming use.
